## Brand & Style

This design system is built for a high-tech, professional environment where precision meets innovation. The aesthetic is rooted in **Modern Corporate** principles with a distinct **Glassmorphic** layer, signaling a future-forward approach to AI solutions. 

The target audience consists of enterprise stakeholders and technical professionals who value reliability, speed, and clarity. The UI evokes a sense of "digital mastery"—calm, authoritative, and sophisticated. Visual interest is generated through deep chromatic depth, utilizing high-contrast typography and subtle translucent surfaces to create a multi-dimensional workspace that feels both airy and grounded.

## Colors

The palette transition moves from a foundational Deep Navy (`#111827`) to a vibrant Electric Blue. The primary color (`#4F46E5`) serves as the core brand driver for actions and primary states, while the secondary color (`#0EA5E9`) provides highlights and accents for data visualization and secondary interactions.

The default mode is **Dark**, optimizing for professional software environments and reducing eye strain. High-contrast white (`#FFFFFF`) is reserved for critical text and primary icons to ensure maximum legibility against the deep backgrounds. Success, warning, and error states should utilize the Electric Blue hue but shifted toward green, yellow, and red respectively, while maintaining the same saturation and brightness levels as the primary blue.

## Typography

This design system utilizes a dual-font strategy. **Plus Jakarta Sans** is employed for display and headlines to provide a modern, slightly geometric character that feels approachable yet professional. **Inter** is used for all body text, labels, and UI elements, chosen for its exceptional legibility in technical contexts and wide-ranging weights.

Hierarchy is established through significant scale jumps and weight differentiation. Headlines use tighter letter spacing to maintain a "lock-up" feel, while labels utilize increased letter spacing for clarity at smaller sizes. For mobile views, display and large headline sizes scale down to prevent excessive line wrapping.

## Layout & Spacing

The layout follows a **Fluid Grid** model with a max-width container for desktop viewing. A 12-column grid is standard for desktop, transitioning to 8 columns for tablets and 4 columns for mobile.

Spacing is based on a **4px baseline rhythm**. Generous whitespace is a core requirement; allow for significant vertical separation (`48px` or `xl`) between major content blocks to emphasize the premium nature of the brand. Internal component padding should favor the `16px` (md) and `24px` (lg) units to create "breathable" interfaces. Containers should align to the grid, but internal layouts for AI-driven dashboards may use flexbox/auto-layout for contextual density.

## Elevation & Depth

Hierarchy is achieved through a combination of **Tonal Layering** and **Glassmorphism**. 

1.  **Base Layer:** The deepest navy background (`#0B0F1A`).
2.  **Surface Layer:** A slightly lighter navy (`#111827`) for secondary containers.
3.  **Glass Layer:** Primary cards use a backdrop-filter (blur: 12px) with a semi-transparent background (`rgba(17, 24, 39, 0.7)`) and a subtle `1px` border (`rgba(255, 255, 255, 0.1)`).
4.  **Shadows:** Shadows are highly diffused and tinted with the primary blue color (e.g., `box-shadow: 0 20px 40px rgba(0, 0, 0, 0.4), 0 0 20px rgba(79, 70, 229, 0.1)`).

This stack creates a sense of "floating" high-tech panels without the heaviness of traditional flat design.

## Shapes

The shape language is refined and balanced. A `0.5rem` (8px) corner radius is the standard for most components (buttons, input fields, chips). Larger containers and cards utilize the `rounded-lg` (16px) or `rounded-xl` (24px) settings to soften the high-tech edge, making the interface feel modern rather than aggressive. 

Interactive elements like buttons should maintain consistent corner radii to reinforce their "clickability" against static background panels.

## Components

### Buttons
Primary buttons use a solid `#4F46E5` fill with white text. Secondary buttons use a glass effect with a subtle border. All buttons have a high-hover state involving a soft outer glow in the electric blue hue.

### Cards
Cards are the primary container for AI insights. They must feature the 12px backdrop blur and the semi-transparent border. Titles within cards use `headline-md`.

### Input Fields
Inputs are dark-themed with a `1px` border that glows primary blue on focus. Use `body-sm` for placeholder text in a muted gray (`#6B7280`).

### Chips & Tags
Chips are used for metadata and status. They should have a low-opacity background tint of the status color (e.g., subtle green for "Success") with a higher-opacity text color.

### Lists
Lists use thin separator lines (`rgba(255, 255, 255, 0.05)`) and should include generous `16px` vertical padding for each item to maintain the "airy" aesthetic.

### Additional Elements
- **Gradients:** Use subtle linear gradients for data visualizations, moving from `#4F46E5` to `#0EA5E9`.
- **Icons:** Use thin-stroke (2pt) icons to match the refined typography of Plus Jakarta Sans.
